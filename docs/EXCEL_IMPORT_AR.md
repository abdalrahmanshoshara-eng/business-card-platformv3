# استيراد ملف Excel القديم إلى قاعدة بيانات Django

## الفكرة

لم يعد ملف Excel هو قاعدة البيانات الأساسية. هذا المشروع يعتمد على جدول `BusinessCard` داخل قاعدة البيانات، ويستخدم Excel فقط كمصدر استيراد أولي أو كملف تصدير للنسخ الاحتياطي.

تمت إضافة أمر Django جديد:

```bash
python manage.py import_cards_excel /path/to/cards.xlsx
```

الأمر يقرأ أوراق ملف Excel القديمة، ينظف القيم، يحولها إلى شكل الموديل، يمنع التكرار، ثم يحفظها في قاعدة البيانات.

## ربط أعمدة Excel مع قاعدة البيانات

| Excel | قاعدة البيانات |
|---|---|
| م | لا يُحفظ كرقم أساسي، فقط داخل `raw_text`، والترقيم الجديد يكون متسلسلاً بدون تكرار |
| اسم الشركة | `company_name`, `company_name_ar`, `company_name_en` |
| اسم الشخص عربي | `person_name_ar` |
| اسم الشخص إنجليزي | `person_name_en` |
| الصفة | `job_title_ar` أو `job_title_en` حسب اللغة |
| رقم الهاتف | `mobile_numbers` كقائمة JSON |
| البريد الإلكتروني | `emails` كقائمة JSON |
| الدولة + المدينة | `address` |
| نوع الاستثمار | `investment_type` إذا كان مطابقاً للقائمة الرسمية، وإلا يستخدم التصنيف التلقائي أو `investment_type_other` |
| طبيعة الاستثمار + ملاحظات | `company_activity` |
| الموقع الإلكتروني | `website` |
| ملاحظات | تدخل ضمن `company_activity` و/أو `review_notes` عند الحاجة |

## التشغيل داخل Docker

ضع الملف داخل:

```text
backend/imports/cards.xlsx
```

ثم شغل:

```bash
docker compose exec backend python manage.py import_cards_excel /app/imports/cards.xlsx --dry-run
```

إذا كانت النتيجة صحيحة، نفذ الاستيراد الفعلي:

```bash
docker compose exec backend python manage.py import_cards_excel /app/imports/cards.xlsx --clear
```

استخدم `--clear` فقط عندما تكون قاعدة البيانات فارغة أو لا يوجد شيء تريد الحفاظ عليه.

## خيارات الأمر

```bash
--dry-run
```
يفحص الملف ويعطي ملخصاً دون أي تعديل على قاعدة البيانات.

```bash
--clear
```
يحذف كل الكروت الموجودة قبل الاستيراد. مناسب عند الانتقال من SQLite إلى PostgreSQL وقاعدة البيانات لا تحتوي بيانات مهمة.

```bash
--update-existing
```
إذا وجد تكراراً بنفس البصمة، يحدث السجل القديم بدلاً من تجاهله.

```bash
--max-rows 20
```
لاستيراد أول 20 صفاً فقط عند التجربة.

## منع التكرار

الأمر يبني بصمة `duplicate_hash` وفق هذا الترتيب:

1. البريد الإلكتروني إن وجد.
2. رقم الهاتف إن لم يوجد بريد.
3. الاسم + الموقع.
4. الاسم + الشركة + الموقع.

هذا يمنع تكرار نفس الشخص أو نفس الكرت عند وجوده في أكثر من ورقة داخل Excel.

## بعد الاستيراد

افتح الداشبورد:

```text
http://IP-اللابتوب:3000/dashboard
```

ثم راجع الفلاتر:

- `needs_review=true` للسجلات التي تحتاج تدقيقاً.
- البحث باسم الشركة أو الشخص أو النشاط.
- التصدير من زر Export Excel إذا أردت نسخة مراجعة بعد الإدخال في قاعدة البيانات.
