# دليل نقل منصّة الكروت إلى سيرفر SSH (كامل: باك + فرونت + قاعدة بيانات + الكروت)

هذا الدليل ينقل مشروعك بالكامل إلى سيرفر لينكس (Ubuntu/Debian) ويشغّله على دومين مع شهادة HTTPS.
مشروعك مُجهّز أصلاً بـ Docker، لذلك كل شيء يعمل داخل حاويات: قاعدة بيانات PostgreSQL، وباك إند Django، وفرونت إند Next.js، وأمامها Nginx يتولّى الدومين وHTTPS.

المكوّنات التي أضفتها لك في مجلد `deploy/`:

| الملف | الوظيفة | يعمل على |
|------|---------|----------|
| `docker-compose.prod.yml` | تشغيل كل الخدمات + Nginx + Certbot | السيرفر |
| `nginx/app.conf.template` | توجيه الطلبات وHTTPS | السيرفر |
| `init-letsencrypt.sh` | إصدار شهادة SSL أول مرة | السيرفر |
| `load-database.sh` | تحميل نسخة قاعدة البيانات | السيرفر |
| `migrate-to-server.sh` | نقل الكود والكروت والداتابيس | جهازك |
| `env-root.example` / `env-backend.example` / `env-frontend.example` | قوالب ملفات البيئة | السيرفر |

---

## قبل أن تبدئي — جهّزي هذه المعلومات

1. **إعدادات SSH** التي أُعطيت لك: اسم المستخدم، IP السيرفر، المنفذ (غالباً 22)، وكلمة المرور أو ملف المفتاح.
2. **الدومين**: مثل `cards.example.com`. يجب أن تملكي صلاحية تعديل سجلّات DNS له.
3. **مفاتيح Gemini** المستخدمة حالياً في `backend/.env` عندك (لتحليل الكروت) — انسخيها لتنقليها.

---

## الخطوة 0 — وجّهي الدومين إلى السيرفر (DNS)

من لوحة مزوّد الدومين، أضيفي سجلّاً من نوع **A**:

```
النوع: A     الاسم: @ (أو cards)     القيمة: IP-السيرفر
النوع: A     الاسم: www              القيمة: IP-السيرفر
```

انتظري بضع دقائق حتى ينتشر. تحقّقي:

```bash
ping example.com     # يجب أن يظهر IP السيرفر
```

> بدون توجيه الدومين، لن تُصدَر شهادة HTTPS في الخطوة 6.

---

## الخطوة 1 — جهّزي السيرفر (تثبيت Docker)

ادخلي على السيرفر:

```bash
ssh المستخدم@IP-السيرفر        # أضيفي -p المنفذ إن لم يكن 22
```

ثبّتي Docker وDocker Compose:

```bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER      # ثم اخرجي وادخلي ثانيةً ليأخذ التأثير
docker --version && docker compose version
```

افتحي المنافذ في الجدار الناري (إن كان مفعّلاً):

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
```

> تأكّدي أيضاً من أن مزوّد السيرفر لا يحجب 80/443 من لوحة التحكم الخاصة به.

---

## الخطوة 2 — انقلي المشروع من جهازك

على **جهازك أنتِ** (وليس السيرفر). على ويندوز استخدمي **Git Bash** أو **WSL**.

افتحي `deploy/migrate-to-server.sh` وعدّلي الأسطر العلوية (المستخدم، IP، المنفذ، المفتاح، ومسار المشروع على السيرفر)، ثم:

```bash
cd مسار/المشروع/business-card-platform
bash deploy/migrate-to-server.sh
```

السكربت يقوم تلقائياً بـ:

1. أخذ نسخة حيّة من قاعدة بياناتك المحلية (أو استخدام `backup_before_reassign.sql` إن كانت الحاوية متوقّفة).
2. رفع الكود إلى السيرفر (بدون `node_modules` و`.venv` و`.next` والأسرار — تُبنى على السيرفر).
3. رفع مجلد **صور الكروت** `backend/media` بالكامل (~129 ميغابايت).
4. رفع ملف نسخة قاعدة البيانات.

> **بديل يدوي** إن لم يتوفّر rsync: استخدمي `scp -r` لرفع المجلد، أو ارفعي الكود عبر Git ثم انقلي `backend/media` ونسخة الداتابيس بـ `scp` فقط.

## الخطوة 3 — جهّزي ملفات البيئة على السيرفر

على السيرفر، داخل مجلد المشروع:

```bash
cd /opt/business-card-platform        # أو المسار الذي اخترتِه

cp deploy/env-root.example     .env
cp deploy/env-backend.example  backend/.env
cp deploy/env-frontend.example frontend/.env.local
```

حرّري `.env` (إعدادات عامة + كلمة مرور قاعدة البيانات + الدومين):

```bash
nano .env
```

عدّلي: `DOMAIN`، `CERTBOT_EMAIL`، و`POSTGRES_PASSWORD` (كلمة مرور قوية).

ثم حرّري `backend/.env`:

```bash
nano backend/.env
```

المهم فيه:
- `DEBUG=false`
- `ALLOWED_HOSTS` يحتوي دومينك
- `DATABASE_URL` بنفس كلمة مرور `POSTGRES_PASSWORD` أعلاه
- `CORS_ALLOWED_ORIGINS` و`CSRF_TRUSTED_ORIGINS` بصيغة `https://دومينك`
- الصقي مفاتيح **Gemini** الحقيقية
- ولّدي `SECRET_KEY` جديداً: `python3 -c "import secrets; print(secrets.token_urlsafe(64))"`

## الخطوة 4 — ابني وشغّلي كل الخدمات

```bash
docker compose -f deploy/docker-compose.prod.yml up -d --build
```

أول مرّة تستغرق دقائق (بناء الصور). تابعي الحالة:

```bash
docker compose -f deploy/docker-compose.prod.yml ps
docker compose -f deploy/docker-compose.prod.yml logs -f backend
```

> ستبدأ الخدمات. Nginx قد يعطي خطأ شهادة الآن لأنها لم تُصدَر بعد — طبيعي، سنعالجه في الخطوة 6.

---

## الخطوة 5 — حمّلي قاعدة البيانات (كل بياناتك)

```bash
bash deploy/load-database.sh db_dump_XXXXXXXX.sql
```

(استبدلي الاسم بالملف الذي رُفع — يظهر في نهاية سكربت النقل.)

هذا يستعيد كل الجداول والبيانات، ثم يطبّق أي هجرات ناقصة. صور الكروت مربوطة بالفعل عبر مجلد `backend/media` المنقول.

لإنشاء حساب إدارة (اختياري):

```bash
docker compose -f deploy/docker-compose.prod.yml exec backend python manage.py createsuperuser
```

---

## الخطوة 6 — أصدري شهادة HTTPS

بعد أن أصبح الدومين موجّهاً للسيرفر (الخطوة 0):

```bash
# جرّبي أولاً بشهادة تجريبية: اجعلي CERTBOT_STAGING=1 في .env
bash deploy/init-letsencrypt.sh
```

إذا نجحت التجربة، أعيدي `CERTBOT_STAGING=0` في `.env` وشغّلي السكربت مرة أخرى للشهادة الحقيقية.

التجديد بعد ذلك **تلقائي** عبر حاوية `certbot` كل 12 ساعة.

افتحي المتصفّح على: **https://دومينك** ✅

## التحقّق النهائي

```bash
# كل الحاويات تعمل؟
docker compose -f deploy/docker-compose.prod.yml ps

# الموقع يستجيب عبر HTTPS؟
curl -I https://دومينك

# واجهة API تعمل؟
curl -I https://دومينك/api/

# صورة كرت تظهر؟ افتحي رابط أي صورة /media/... في المتصفّح
```

اختبري: تسجيل الدخول، ظهور الكروت وصورها، إضافة كرت جديد، ولوحة الإدارة على `/admin/`.

## أوامر تشغيل يومية مفيدة

```bash
C="docker compose -f deploy/docker-compose.prod.yml"

$C ps                      # حالة الخدمات
$C logs -f backend         # سجلّات الباك
$C restart backend         # إعادة تشغيل خدمة
$C down                    # إيقاف الكل (البيانات تبقى محفوظة)
$C up -d --build           # إعادة البناء بعد تحديث الكود

# نسخة احتياطية دورية لقاعدة البيانات:
$C exec -T postgres pg_dump -U business_cards -d business_cards --clean --if-exists > backup_$(date +%F).sql
```

## نقاط مهمّة للأمان

- **DEBUG=false** إجباري على دومين عام (عالجناه بجعل Nginx يخدّم صور الكروت مباشرة).
- غيّري كلمة مرور قاعدة البيانات الافتراضية `business_cards` إلى كلمة قوية.
- منفذ قاعدة البيانات غير مكشوف للإنترنت (مربوط بـ 127.0.0.1 فقط).
- لا ترفعي ملفات `.env` الحقيقية إلى Git — القوالب فقط.
- خذي نسخة احتياطية دورية لقاعدة البيانات ولمجلد `backend/media`.

## إن واجهتِ مشكلة

| العرض | السبب المحتمل | الحل |
|------|----------------|------|
| فشل إصدار الشهادة | الدومين لم يُوجّه بعد / 80 محجوب | تأكّدي من DNS ومن فتح منفذ 80 |
| صفحة 502 Bad Gateway | خدمة لم تُقلع | `logs -f frontend` و`logs -f backend` |
| صور الكروت لا تظهر | مجلد media لم يُرفع | تأكّدي من محتوى `backend/media` على السيرفر |
| خطأ CSRF عند الدخول | نطاقات غير مضبوطة | راجعي `CSRF_TRUSTED_ORIGINS` بصيغة https |
| قاعدة بيانات فارغة | لم تُحمّل النسخة | أعيدي الخطوة 5 |
