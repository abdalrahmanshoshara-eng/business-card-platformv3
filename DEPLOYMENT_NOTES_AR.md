# ملاحظات نشر منصة الكروت على السيرفر

## الوضع الحالي
- السيرفر: `wasim@91.144.45.21` عبر SSH على المنفذ **2222**.
- عنوان السيرفر الداخلي (خلف NAT): **192.168.11.5**
- المشروع على السيرفر في: `~/app`
- كل شيء يعمل داخل Docker: PostgreSQL + Django (backend) + Next.js (frontend).
- البيانات المنقولة: **283 كرت** + صور الكروت (مجلد `backend/media`).
- الفرونت منشور على المنفذين **3000** و**8080** داخلياً.

## الوصول للمنصة

### 1) بعد أن يوجّه مدير الشبكة منفذاً (الحل الدائم)
رسالة لمدير الشبكة:
> أرجو عمل Port Forwarding: منفذ عام TCP (مثل 80 أو 8080) → إلى الجهاز الداخلي `192.168.11.5` على المنفذ `3000` — تماماً كما هو موجَّه SSH 2222.

بعدها تُفتح المنصة على: `http://91.144.45.21:<المنفذ>`
(الإعداد جاهز لقبول 80 و8080 و3000.)

### 2) الآن فوراً — أنتِ من أي مكان (نفق SSH عبر PuTTY)
- جلسة PuTTY إلى `91.144.45.21` منفذ `2222`.
- Connection → SSH → Tunnels: Source `8080`، Destination `localhost:3000` → Add → Open.
- افتحي في المتصفّح: `http://localhost:8080`

### 3) أي جهاز على نفس شبكة السيرفر المحلية
- `http://192.168.11.5:3000`

## أوامر الصيانة (على السيرفر داخل `~/app`)
```bash
docker compose ps                      # حالة الخدمات
docker compose logs -f backend         # سجلّات الباك إند
docker compose logs -f frontend        # سجلّات الفرونت
docker compose restart backend         # إعادة تشغيل خدمة
docker compose stop                    # إيقاف الكل (البيانات تبقى)
docker compose up -d                   # تشغيل الكل
docker compose up -d --build           # إعادة البناء بعد تحديث الكود (git pull)
```

## نسخة احتياطية لقاعدة البيانات
```bash
cd ~/app
docker compose exec -T postgres pg_dump -U business_cards -d business_cards --clean --if-exists > backup_$(date +%F).sql
```
واحتفظي أيضاً بنسخة من مجلد `~/app/backend/media` (صور الكروت).

## ملاحظات تقنية مهمة
- شبكة السيرفر تحجب `pypi.org` (DNS)، لذلك أُضيف سطر في `/etc/hosts` يوجّهه لعنوان Fastly، ومُرِّر لعملية البناء عبر `docker-compose.override.yml` (extra_hosts). لا تحذفي هذا وإلا فشل بناء الباك إند.
- خدمات الأنفاق الخارجية (Cloudflare, ngrok, Tailscale) محجوبة من موقع السيرفر (بعضها حجب قانوني 451)، لذا الوصول العام يكون عبر توجيه منفذ من مدير الشبكة فقط.
- `DEBUG=true` حالياً (كي تُخدَّم صور الكروت). لبيئة إنتاج عامة صارمة يُفضّل لاحقاً `DEBUG=false` مع خدمة الميديا عبر خادم ويب.

## أمان
- غيّري كلمة مرور قاعدة البيانات الافتراضية (`business_cards`) لكلمة قوية عند الحاجة (في `.env` جذر المشروع وفي `backend/.env`).
- ألغي أي مفتاح Tailscale ظهر سابقاً من صفحة Keys.
- ملفات النقل المؤقتة (`~/app/db_dump.sql`, `~/app/media.tar.gz`, `~/tmp-data`) يمكن حذفها بعد التأكد أن كل شيء يعمل.
